# Product-as-Character Workflow

Products (devices, packaging, vehicles, branded items) are treated as characters in the production pipeline. They get turnarounds and variations just like human characters.

## Product Turnaround

Same 2x2 grid format as character turnarounds, adapted for products:
- Top left: Front view (logo/brand face visible)
- Top right: 3/4 angle (showing depth and form factor)
- Bottom left: Side profile (thickness, ports, buttons)
- Bottom right: Back view (rear features, labels, connections)

Key differences from character turnarounds:
- Emphasize material textures (brushed aluminum, matte plastic, glass, fabric)
- Consistent studio lighting that reveals surface properties
- Logo and brand elements clearly visible in relevant angles
- Scale reference when size matters for downstream shots

## Product Faithfulness

Faithfully replicate based on script character attributes and any reference images. Avoid creative interpretations or imaginative adaptations unless user specifies.

Rules:
- Match colors, proportions, and design language exactly
- Do NOT invent features not described in script or visible in references
- Do NOT add decorative elements, custom branding, or stylistic embellishments
- Use neutral descriptors ("the white device") rather than specific model names in prompts

## Product Variations

After establishing turnaround, variations show the product in context:
- Feature highlight shots (close-up on key feature, dramatic lighting)
- Usage scenario (product in natural environment, being used)
- Detail shots (texture close-up, material quality, craftsmanship)
- Hero shots (dramatic angle, premium lighting, aspirational setting)

Each variation should maintain exact product appearance while varying only the context, angle, and lighting treatment.

## Tool Selection for Products

- **Nano Banana**: Always default for products. Precision reference matching is essential for brand accuracy. Handles multiple reference images for complex products.
